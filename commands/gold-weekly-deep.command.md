# 黄金周度深度分析

直接调用 **黄金深度分析器（gold-deep-analyzer）**，执行专家级深度研判。

## 执行流程

```
用户执行 /gold-weekly-deep
        ↓
Step 1: 检查并采集最新数据
        ↓
Step 2: 激活 gold-deep-analyzer Agent
        ↓
Step 3: 执行深度分析四阶段
        ├─ Phase 1: Deep Research 问题分解
        ├─ Phase 2: Self-Reflecting Chain 七维分析
        ├─ Phase 3: Complex Reasoning 综合推理
        └─ Phase 4: 生成完整报告
        ↓
Step 4: 输出并保存结果
```

## 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `--period` | string | 本周 | 分析周期（本周/上周/YYYY-MM-DD格式） |
| `--focus` | string | all | 重点关注维度（all/price/macro/policy/seasonal） |
| `--output` | string | both | 输出方式（console/file/both） |
| `--save_memory` | boolean | true | 是否保存到长期记忆 |

## 使用示例

```bash
# 默认执行（分析本周，全维度，输出到控制台和文件）
/gold-weekly-deep

# 重点关注政策维度（适合FOMC周）
/gold-weekly-deep --focus policy

# 重点关注季节性维度
/gold-weekly-deep --focus seasonal

# 仅输出到控制台
/gold-weekly-deep --output console

# 分析上周数据
/gold-weekly-deep --period 上周
```

## 深度分析四阶段

### Phase 1: Deep Research - 问题分解与规划

#### 1.1 认知偏差预检（关键新增）

执行 `/gold-weekly-deep` 前，自动进行认知偏差自检：

```markdown
## 认知偏差预检清单

□ **确认偏误检查**
  - 我是否只关注支持我看涨/看跌的证据？
  - 我是否主动寻找了反方观点？

□ **锚定效应检查**
  - 我是否被近期价格（如上周收盘价）过度锚定？
  - 我是否被某个"关键价位"限制了思维？

□ **过度自信检查**
  - 我的预测是否过于精确（如预测到具体点位）？
  - 我是否为预测错误预留了足够空间？

□ **损失厌恶检查**
  - 我是否因为不想承认错误而坚持原有观点？
  - 我是否过早获利了结？

□ **从众心理检查**
  - 我的观点是否只是市场共识的重复？
  - 我是否有独立的信息来源和逻辑？

**偏差纠正措施**:
- 强制写出"反方观点"
- 使用概率区间而非点估计
- 设置24小时冷静期（重大决策）
```

#### 1.2 深度分析规划

**分析目标**: 本周黄金市场深度研判

**子任务分解**:
1. 价格维度分析 - 技术走势、支撑阻力
2. 宏观维度分析 - 美元、利率、通胀
3. 商品维度分析 - 原油、金油比
4. 情绪维度分析 - VIX、RSI、市场情绪
5. 政策维度分析 - 美联储、央行动态
6. 地缘维度分析 - 地缘政治风险
7. 季节性维度分析 - 当前周期阶段、历史规律

**数据来源**:
- `data/today_data.json` - 实时市场数据
- `data/gold_tracker_cache.json` - 历史缓存
- `memory/framework.md` - 分析框架
- Web搜索 - 补充宏观/政策信息

**整体置信度目标**: ≥70%（任一维度<60%需回溯）

### Phase 2: Self-Reflecting Chain - 七维逐步推理

每个维度执行以下标准化流程：

```markdown
## Step [N]: [维度名称]分析

### 执行
[执行该维度的具体分析]

### 数据依据
- 指标1: [数值] (来源: [api/cache/search])
- 指标2: [数值] (来源: [api/cache/search])

### 分析结果
[该维度的分析结论]

### 自我反思（关键步骤）
- **置信度**: [0-100]%（参考评估标准）
- **假设**: [本分析基于哪些假设]
- **逻辑检查**: [推理是否严密，是否存在跳跃]
- **潜在错误**: [哪些因素可能导致分析错误]
- **反方观点**: [如果分析错误，最可能的原因是什么]

### 置信度评估标准
| 置信度 | 含义 | 决策 |
|--------|------|------|
| 90-100% | 高度确信，数据充分，逻辑严密 | ✅ 继续 |
| 80-89% | 较为确信，少量假设 | ✅ 继续 |
| 70-79% | 基本确信，存在一定不确定性 | ✅ 继续 |
| 60-69% | 低置信度，假设较多 | ⚠️ 低置信度但继续（标注风险） |
| <60% | 置信度不足，需要回溯修正 | ❌ 回溯 |

### 决策
- ✅ **继续** (置信度≥70%)
- ⚠️ **低置信度但继续** (60-69%，需标注风险)
- ❌ **回溯** (置信度<60%，需补充数据或修正逻辑)
```

## 最佳实践

1. **FOMC周**: 使用 `--focus policy` 重点关注政策维度
2. **季节性转折**: 使用 `--focus seasonal` 深入分析周期阶段
3. **突破行情**: 使用 `--focus price` 重点技术分析
4. **定期回顾**: 每周一执行 `/gold-weekly-deep` 生成周度报告
