# 生成黄金报告

根据采集的市场数据，生成专业的黄金跟踪报告。

## 报告格式自动选择规则

| 条件 | 自动选择格式 | 说明 |
|------|-------------|------|
| 周一 | 格式C（周度深度） | 全量分析+下周展望 |
| FOMC前后 | 格式B（事件快报） | 聚焦美联储决策影响 |
| 三级预警触发 | 格式B（事件快报） | 紧急风险提示 |
| 叙事切换 | 格式B（事件快报） | 说明逻辑变化 |
| 9月/11月首次 | 格式B（事件快报） | 季节性提醒 |
| 其他工作日 | 格式A（简报） | 3行核心信息 |
| 周末 | 不生成 | 跳过 |

## 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `--format` | string | auto | 报告格式（A/B/C/auto） |
| `--output` | string | console | 输出方式（console/file/both） |

## 使用示例

```bash
# 自动生成（推荐）
/gold-report

# 强制生成简报
/gold-report --format A

# 生成事件快报
/gold-report --format B

# 生成周度深度报告（自动调用深度分析Agent）
/gold-report --format C

# 生成深度报告并保存到文件
/gold-report --format C --output file

# 同时输出到控制台和文件
/gold-report --format C --output both
```

## 报告内容结构

### 格式A：简报（3行）
- 日期和数据概览
- 核心价格指标
- 操作建议（50字内）

### 格式B：事件快报
- 核心数据（仅变化项）
- 当前叙事状态
- 事件驱动分析
- 季节性提醒（如适用）
- 操作建议（100字内）

### 格式C：周度深度报告

**⚡ 此格式将自动调用黄金深度分析器（gold-deep-analyzer）**

深度分析流程：
1. **Phase 1**: 问题分解与规划（Deep Research）
2. **Phase 2**: 七维逐步推理与验证（Self-Reflecting Chain）
   - 维度1: 价格维度
   - 维度2: 宏观维度
   - 维度3: 商品维度
   - 维度4: 情绪维度
   - 维度5: 政策维度
   - 维度6: 地缘维度
   - 维度7: 季节性维度
3. **Phase 3**: 综合推理（Complex Reasoning）
   - Tree-of-Thought策略选择
   - MECE风险识别
4. **Phase 4**: 输出完整深度分析报告

报告内容包括：
- 执行摘要（含整体置信度）
- 七维深度分析（每维度含置信度）
- 策略建议（Tree-of-Thought比较）
- 风险提示（MECE分类）
- 康波周期背景
- 季节性规律研判

## 格式C深度分析执行流程

当用户执行 `/gold-report --format C` 时：

```bash
1. 检查数据完整性
   └─ 若无今日数据，先执行 /gold-fetch

2. 激活 gold-deep-analyzer Agent
   └─ 触发关键词: "深度分析本周黄金市场"

3. Agent执行七维深度分析
   ├─ Phase 1: 问题分解（Deep Research）
   ├─ Phase 2: 逐步推理（Self-Reflecting Chain）
   │   ├─ 维度1: 价格维度分析 + 置信度评估
   │   ├─ 维度2: 宏观维度分析 + 置信度评估
   │   ├─ 维度3: 商品维度分析 + 置信度评估
   │   ├─ 维度4: 情绪维度分析 + 置信度评估
   │   ├─ 维度5: 政策维度分析 + 置信度评估
   │   ├─ 维度6: 地缘维度分析 + 置信度评估
   │   └─ 维度7: 季节性维度分析 + 置信度评估
   ├─ Phase 3: 综合推理（Complex Reasoning）
   │   ├─ Tree-of-Thought: 多策略比较
   │   └─ MECE分析: 风险识别
   └─ Phase 4: 生成完整报告

4. 输出报告
   ├─ console: 显示在终端
   └─ file: 保存到 memory/YYYY-MM-DD-deep.md

5. 返回执行结果
   └─ 报告路径 + 整体置信度 + 推荐策略
```
